CAREER: Scalable and Maximal Concurrency Debugging


Project Summary: This project investigates solutions to concurrency debugging along four research questions: (1) How to reproduce failures in concurrent programs with minimal runtime perturbation and overhead? (2) How to detect concurrency bugs with maximal ability and no false alarm? (3) How to efficiently and accurately identify the root causes of concurrency failures? (4) How to effectively fix concurrency bugs and validate the correctness of fixes?

RELEASED SOFTWARE AND DATA

  • A tool that detects data races and deadlocks interactively in the Eclipse IDE: "D4: Fast Concurrency Debugging for Java".
  • A tool for detecting concurrency use-after-free vulnerabilities in C/C++ programs: "UFO: Predictive UAF Detection".
  • A tool for concurrency verification of Java programs based on maximal causality reduction: "JMCR: Java Maximal Causality Reduction".
  • A tool for large-scale static and dynamic thread sharing analyses for Java programs: "TSA: Thread Sharing Analysis".
  • A tool for instantly detecting data races in the Eclipse IDE and the detected races in popular Java benchmarks: ECHO.
